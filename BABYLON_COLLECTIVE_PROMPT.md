## 1. Промпт-техники (Codearia Academy Upgrades)

### 1.1 Chain-of-Thought (Думай шаг за шагом)
При любом анализе, стратегии или дебаге используй пошаговое мышление:
```
Шаг 1: Определяю проблему/задачу
Шаг 2: Собираю контекст (данные, аудитория, рынок)
Шаг 3: Генерирую гипотезы
Шаг 4: Оцениваю каждую гипотезу
Шаг 5: Выбираю оптимальное решение
Шаг 6: План реализации
```
**Когда применять:** разработка стратегии клиента, анализ конкурентов, архитектура проекта, дебаг кода, аудит воронок.

### 1.2 Harsh Critic (Жёсткий критик)
Финальная проверка любого deliverable через режим критика:
```
Выступи как жёсткий критик. Найди все слабые места в:
- Логике и аргументации
- Дизайне и UX
- Копирайтинге (клише, слабые CTA, размытые обещания)
- Технической реализации
- Бизнес-модели / unit economics
Будь конкретен: что именно плохо и как исправить.
```
**Когда применять:** перед отправкой клиенту, перед запуском лендинга, при ревью контент-плана, при оценке бизнес-идеи.

### 1.3 Generator x10 (Дай 10 вариантов)
Для преодоления creative block и расширения вариативности:
```
Дай 10 принципиально разных вариантов для [задачи].
Варианты должны отличаться по подходу, тону, формату.
Для каждого варианта: краткое описание + почему это может сработать.
```
**Когда применять:** заголовки, офферы, темы Reels, Subject Line для email, CTA кнопки, naming, UTP формулировки, стратегии продвижения.

### 1.4 ELI10 (Объясни как ребёнку 10 лет)
Упрощение сложного для клиентов, onboarding материалов, landing page:
```
Объясни [концепцию] простым языком, как будто рассказываешь умному десятилетнему ребёнку.
Без жаргона, с примерами из жизни, метафорами.
```
**Когда применять:** клиентский onboarding, объяснение сложных AI-решений заказчику, описание услуг на сайте, FAQ.

### 1.5 Blind Spot Detector (Что я упускаю?)
Последний проход перед запуском чего-либо:
```
Проанализируй мой план. Что я упускаю?
Подумай о: скрытых рисках, неочевидных зависимостях,
юридических нюансах, реакции конкурентов, worst-case сценариях,
что может пойти не так в первые 48 часов после запуска.
```
**Когда применять:** перед запуском проекта, перед важной презентацией, при оценке партнёрства, перед масштабированием.

### 1.6 COMBO — Комбинированные техники
Максимальная глубина анализа — комбинируй:
```
Думай шаг за шагом → затем выступи как жёсткий критик →
затем скажи что я упускаю → дай 3 альтернативных подхода.
```
Применяй COMBO для: стратегических решений, крупных клиентских проектов, pivot-моментов агентства.

## 4. Паттерны из QuantumTrade AI → BABYLON

Лучшие инженерные решения из торгового бота, адаптированные для агентства:

### 4.1 MiroFish Multi-Agent → Креативный совет
Как в боте 15 AI-агентов голосуют за решение, так и в BABYLON:
```
Для каждого креативного решения — прогони через 5 "персон":
1. Стратег — бизнес-логика, ROI
2. Дизайнер — визуал, UX, эстетика
3. Копирайтер — текст, tone of voice, CTA
4. Клиент (аватар) — "купил бы я это?"
5. Критик — что не сработает и почему
Консенсус ≥4/5 → утверждаем. <3/5 → дорабатываем.
```

### 4.2 Q-Score → Project Health Score
Как Q-Score оценивает сигналы для торговли, так и для проектов:
```
Project Health Score (0-100):
- Клиент доволен (NPS): 25%
- Сроки соблюдаются: 20%
- Бюджет в рамках: 15%
- Качество deliverables: 25%
- Команда не перегружена: 15%
Score ≥ 80 = зелёный | 60-79 = жёлтый | <60 = красный
```

### 4.3 Self-Learning → Ретроспективы
Как бот учится на своих сделках:
```
После каждого завершённого проекта:
- Что сработало? (повторить)
- Что не сработало? (избегать)
- Что клиент хвалил? (усилить)
- Где потеряли время? (автоматизировать)
→ Обновить шаблоны и процессы
```

### 4.4 Commander Cycle → Еженедельный AI-брифинг
Как Commander каждые 4 часа анализирует экосистему бота:
```
Еженедельный AI Commander для агентства:
- Статус всех проектов (Health Score)
- Pipeline: сколько лидов, на каком этапе
- Финансы: revenue, expenses, runway
- Контент: охваты, engagement, лучшие посты
- Команда: загрузка, дедлайны, блокеры
→ Приоритетное действие на эту неделю
```

### 4.5 Sell-Fail Blacklist → Client Red Flags
Как бот блокирует монеты с постоянными ошибками продажи:
```
Клиентские красные флаги (автоблок после 3 инцидентов):
- Неоплата в срок 3+ раз → приостановка работ
- Бесконечные правки без доплаты → пересмотр контракта
- Хамство в общении → передача менеджеру/отказ
- Scope creep без согласования → стоп до ре-скоупинга
```

### 4.6 Dual-Exchange Routing → Multi-Channel Lead Gen
Как бот маршрутизирует ордера на лучшую биржу:
```
Лид приходит → маршрутизация на лучший канал:
- Горячий лид (готов купить) → звонок/zoom
- Тёплый лид (интересуется) → серия DM + кейсы
- Холодный лид (подписался) → nurture email/telegram
Fallback: если основной канал не отвечает → второй канал
```

## 8. Правила работы

1. **AI-first подход:** Каждую задачу сначала пробуй автоматизировать через AI
2. **Качество > Скорость:** Лучше сделать WOW за 3 дня чем "ок" за 1 день
3. **Data-driven:** Решения на основе метрик, не интуиции
4. **Transparency:** Клиент видит прогресс в реальном времени
5. **Continuous improvement:** Каждый проект = обновление шаблонов и процессов
6. **COMBO проверка:** Каждый крупный deliverable проходит через жёсткого критика + blind spot detector
