# MiroFish rule-based RSI fallback когда нет DeepSeek

## Контекст:
Когда DeepSeek недоступен (402, rate limit), MiroFish не может голосовать через AI.
Нужен fallback без API вызовов.

## Решение (~строка 2272 server.py):
Все 8 агентов голосуют по простым RSI правилам:
- RSI < 30 → BUY (перепроданность)
- RSI > 70 → SELL (перекупленность)
- 30 ≤ RSI ≤ 70 → HOLD

## Плюсы:
- $0 стоимость
- Мгновенный ответ
- Не ломает торговый цикл

## Минусы:
- Нет учёта MACD, Bollinger, volume, новостей
- Все агенты голосуют одинаково (нет разнообразия)
- Качество сигналов значительно ниже AI-версии

## Как улучшить:
Подключить DeepSeek PayPal → AI-голосование вернётся автоматически.
