### v10.19.2 — Yield-based estimation (ключевой фикс Earn)
ByBit API bug: `amount: "0"` для активных Simple Earn позиций, но `yesterdayYield > 0`.

Решение: Principal ≈ yesterdayYield × 365 / APR

```python
_est_principal = round(_yield_day * 365 / 0.0206, 2)
```

APR = 2.06% — подтверждён эмпирически: $0.00113/день × 365 / $20 = 2.06%

Результат: earn_total показывает $127.38 ($107.35 KC + $20.03 BB)

### v10.19.3 — DCI capital fix (двойное исправление)

**Проблема:** DCI выдавал "Amount out of range, min: 20, max: 500000"

**Причина #1:** В `dci_auto_place_idle()` Step 3.6 — when FUND достаточен, `usdt_free` оставался равным UNIFIED ($19.82), а не FUND ($20.50). DCI дебетует с FUND, поэтому capital должен отражать FUND.

Фикс:
```python
else:
    log_activity(f"[dci] FUND=${fund_usdt:.2f} sufficient, no transfer needed")
    usdt_free = fund_usdt  # v10.19.3: DCI debits from FUND
```

**Причина #2:** `str(round(20.0, 2))` → `"20.0"` (Python убирает trailing zero). ByBit strict validation ожидает "20.00".

Фикс в `bybit_dci_place_order`:
```python
"amount": f"{amount:.2f}",  # "20.00" не "20.0"
```

## Технические открытия

### ByBit API особенности
- `/v5/earn/position` всегда возвращает `amount: "0"` для Simple Earn (даже при активных позициях)
- `yesterdayYield` работает правильно — через него можно оценить principal
- INVESTMENT accountType — ByBit объединил типы аккаунтов, реально ничего не возвращает
- FUND account — отдельный от UNIFIED, только для депозитов/выводов/DCI
- DCI: ордера дебетуются с FUND, поэтому UNIFIED баланс не подходит для capital check

### Формат чисел в ByBit API
- ByBit требует строгого формата с двумя десятичными: "20.00" а не "20.0"
- `str(round(x, 2))` даёт "20.0", нужно `f"{x:.2f}"` для "20.00"
