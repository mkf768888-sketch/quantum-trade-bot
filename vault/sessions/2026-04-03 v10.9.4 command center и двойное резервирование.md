# Сессия 2026-04-03: v10.9.4 — Исправление двойного резервирования + Command Center

## Проблема дня
Бот показывал 0 сделок за 97 часов при активном Autopilot. Спот баланс $1.49.

## Найденные и исправленные баги

### 🔴 КРИТИЧЕСКИЙ: Двойное резервирование (0 сделок 97ч)
- **Причина**: Smart Money Router оставлял в споте $1(arb) + $2(trade) = $3
- Trade cycle ТОЖЕ вычитал те же $3 → `tradeable = $0` → никаких сделок
- **Фикс**: trade cycle теперь вычитает только `ARB_RESERVE_USDT ($3)`, не резервы Router'а
- Файл: `server.py` → функция `auto_trade_cycle` → `_kc_tradeable`

### 🔴 КРИТИЧЕСКИЙ: Trade size всегда $2 (ниже минимума $5)
- **Причина**: `_calc_trade_size` ограничивал размер до `ROUTER_TRADE_RESERVE = $2`
- $2 < SPOT_BUY_MIN_USDT ($5) → все сделки блокировались
- **Фикс**: убрали cap, добавили floor: если 8% < $5, используем $5 (28% от $18 — в рамках trading.md)
- `ROUTER_TRADE_RESERVE` поднят $2 → $20 (чтобы Router оставлял достаточно в споте)

### 🟡 ArB → застрявшие монеты (зафиксировано в v10.9.3)
- AR, ICX, SNX зависли после провала leg2/leg3
- ARB_EXEC_ENABLED и XARB_ENABLED = hardcoded False
- Включать только при капитале > $500

### 🟡 Command Center (/command) — AbortSignal
- `AbortSignal.timeout()` не работает в Telegram WebView / iOS Safari <15.4
- **Фикс**: заменили на `AbortController` + `setTimeout` — работает везде

### 🟡 DEGRADED в System Map
- Arb (намеренно выключен) вызывал "DEGRADED"
- **Фикс**: Arb помечен как опциональный — теперь показывает "OPERATIONAL"
