# Сессия 2026-04-05 — v10.11.1: DCI priority + Wave 2A

## Что сделали

### Диагностика DCI
- DCI видит продукты с APY=985% но не размещал
- Причина: BB=$11 полностью зарезервирован (trade=$10 + arb=$1)
- Earn перехватывал USDT раньше DCI
- API ошибки проглатывались молча — в /dciplace показывалось "Причина:" без текста

### v10.11.1 — задеплоено (коммит a1340cd)

**БАГ #1 FIX**: DCI теперь запускается ПЕРВЫМ в earn_monitor_loop, до Earn
- Был: `EARN_ENABLED → earn_auto_place_idle → DCI`
- Стал: `DCI first → EARN_ENABLED → earn_auto_place_idle`
- Дополнительно: DCI теперь работает независимо от EARN_ENABLED

**БАГ #2 FIX**: Ошибки DCI API теперь передаются в ответ /dciplace
- Было: `placed=[]` без reason → "Причина:" пустая строка
- Стало: `reason: "API error: ..."` → видно что пошло не так

**NEW: /balance** — полный расклад KuCoin + ByBit:
- Всего, В Earn, В DCI, Idle (доступно)
- Процент "работает на тебя"

**NEW: таймер в /dci** — "следующий авто-цикл через ~N мин"

**NEW: Force DCI в Mini App** — подключена к реальному /api/dci/place
**NEW: /api/dci/status** — endpoint для Mini App

**Wave 2A**:
- BUY требует ≥2 TA подтверждений (MACD/BB/Stoch/ADX/OBV)
- Per-symbol cooldown 4 часа между покупками одной монеты

### Наблюдения по рынку

- После перевода: /dciplace должен стрелять
