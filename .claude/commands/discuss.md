# /discuss — Обсуждение и планирование (Conversational Mode)
> Как GSD `/gsd discuss` — режим обсуждения без выполнения кода

## Назначение
Стратегическое обсуждение архитектуры, направлений развития, торговых стратегий.
НЕ вносит изменения в код. Только анализ и рекомендации.

## Контекст для загрузки
- CLAUDE.md (архитектура)
- STATE.md (текущее состояние)
- ROADMAP.md (план развития)
- REQUIREMENTS.md (требования)

## Примеры использования
- `/discuss earn` — обсудить стратегию Earn
- `/discuss risk` — обсудить параметры риска
- `/discuss architecture` — обсудить архитектурные решения
- `/discuss polymarket` — обсудить интеграцию Polymarket
- `/discuss profit` — проанализировать пути к прибыльности

## Формат вывода
Структурированный анализ с:
- Текущее состояние
- Варианты решения (pros/cons)
- Рекомендация
- Следующие шаги (без выполнения)
