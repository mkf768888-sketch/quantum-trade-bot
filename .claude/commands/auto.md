# /auto — Полный автономный режим (GSD Wave Execution)
> Адаптация GSD v2 wave architecture для QuantumTrade AI

## Режим работы
Автономное выполнение всех задач из текущего ROADMAP.md волнами.
Каждая волна = один контекстный цикл (~200k tokens).
Между волнами — запись состояния в STATE.md (crash recovery).

## Протокол запуска

### 1. Загрузка контекста
```
Прочитать: CLAUDE.md → STATE.md → ROADMAP.md → REQUIREMENTS.md
Прочитать: .claude/rules/security.md + trading.md
Определить: текущую волну, незавершённые задачи
```

### 2. Wave Planning (Планирование волны)
- Собрать все задачи со статусом `[ ]` из ROADMAP.md
- Группировать по зависимостям (параллельные vs последовательные)
- Оценить сложность: S (1 волна), M (2-3 волны), L (3+ волн)
- Выбрать задачи для текущей волны (не больше 3-5 за раз)

### 3. Wave Execution
Для каждой задачи:
1. **Research** — прочитать код, понять текущее состояние
2. **Plan** — определить файлы для изменения, написать план
3. **Execute** — внести изменения
4. **Verify** — syntax check, test, убедиться что не сломано
5. **Commit** — атомарный коммит с описанием

### 4. Wave Completion
- Обновить ROADMAP.md: `[ ]` → `[x]` для выполненных
- Обновить STATE.md: текущее состояние, что сделано
- Записать: что осталось для следующей волны

## Safe Mode интеграция
- **Перед деструктивными изменениями** → записать точку восстановления в STATE.md
- **Перед деплоем** → запустить security-auditor агент
- **При ошибке** → НЕ продолжать, записать ошибку в STATE.md, ждать

## Ограничения автономного режима
- НЕ деплоить без явного разрешения пользователя
- НЕ менять параметры риска (.claude/rules/trading.md)
- НЕ удалять существующий функционал
- НЕ коммитить секреты
- Логировать каждое действие в STATE.md

## Команда
```
/auto [направление]
```
Примеры:
- `/auto` — все направления по ROADMAP
- `/auto earn` — только Earn модуль
- `/auto polymarket` — только Polymarket
- `/auto design` — только дизайн-система
- `/auto full` — полный цикл всех направлений
